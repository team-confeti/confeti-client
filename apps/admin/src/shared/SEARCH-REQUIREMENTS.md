# 검색 (Search) — Requirements

> 공연명, 장소로 검색하여 대기 목록/페스티벌/콘서트 페이지의 결과를 실시간 필터링한다. 여러 도메인에 걸쳐 공통으로 동작하는 기능.

**Figma**: [Admin 명세 — 검색 결과 화면](https://www.figma.com/design/LcVY1m4kQmMVDGaIPuiJJy/Admin-%EB%AA%85%EC%84%B8?node-id=18-520)

---

## User Flow

1. 사용자가 헤더의 검색바에 공연명 또는 장소를 입력
2. **실시간**으로 검색 결과 필터링 (엔터 없이 즉시 반영)
3. 현재 보고 있는 페이지의 데이터가 검색어에 맞게 필터링됨
4. 검색 결과 항목 클릭 시 → 정보 입력/수정 페이지로 이동
5. 검색어 삭제 시 → 전체 목록으로 복원
6. 오탈자 교정 불필요 (단순 문자열 매칭)

---

## 적용 범위

| 페이지    | 경로         | 검색 대상    | 필터링 항목                    |
| --------- | ------------ | ------------ | ------------------------------ |
| 대기 목록 | `/pending`   | 공연명, 장소 | 대기 중인 공연 리스트          |
| 페스티벌  | `/festival`  | 공연명, 장소 | 진행 예정/종료된 페스티벌 카드 |
| 콘서트    | `/concert`   | 공연명, 장소 | 진행 예정/종료된 콘서트 카드   |
| 대시보드  | `/dashboard` | —            | 검색 동작 안 함                |

---

## 화면 구성

### 검색 결과 표시

- 각 페이지의 기존 UI를 그대로 사용하되, 검색어에 맞는 항목만 필터링
- 페스티벌/콘서트: 공연 포스터, 제목, 부제목, 날짜, 장소 카드로 표시
- 진행 예정/진행 중 vs 종료된 공연 분류는 검색 결과에서도 유지
- 검색 결과가 없으면 빈 상태 UI 표시

## 상태별 처리

| 상태      | 조건           | 표시                 |
| --------- | -------------- | -------------------- |
| 미입력    | 검색어 없음    | 전체 목록 표시       |
| 검색 중   | 검색어 입력됨  | 필터링된 결과만 표시 |
| 결과 없음 | 매칭 항목 없음 | 빈 상태 UI           |

## API 연동

### 공연 검색 (헤더 검색바)

대기목록/페스티벌/콘서트 각 목록 조회 API를 통해 받아온 데이터를 클라이언트 사이드에서 필터링한다.

| Method | Endpoint                        | 설명                      | END_POINT 상수            |
| ------ | ------------------------------- | ------------------------- | ------------------------- |
| GET    | `/admin/performances/drafts`    | 대기 공연 목록 조회       | `END_POINT.GET_DRAFTS`    |
| GET    | `/admin/performances/festivals` | 등록된 페스티벌 목록 조회 | `END_POINT.GET_FESTIVALS` |
| GET    | `/admin/performances/concerts`  | 등록된 콘서트 목록 조회   | `END_POINT.GET_CONCERTS`  |

### 아티스트 검색 (performance-editor 내 라인업 추가용)

| Method | Endpoint                | 설명          | END_POINT 상수                |
| ------ | ----------------------- | ------------- | ----------------------------- |
| GET    | `/admin/artists/search` | 아티스트 검색 | `END_POINT.GET_ARTIST_SEARCH` |

- Query key: `ARTIST_QUERY_KEY.SEARCH(term)`

### 현재 구현 상태

#### 되어 있는 것

- `Layout`에서 `searchQuery` 상태 관리 (`useState`)
- `Header`에 검색 input 렌더 (placeholder: "공연명, 장소 검색...")
- `<Outlet context={{ searchQuery }}>` 로 하위 페이지에 전달

#### 안 되어 있는 것

- **어떤 페이지도 `searchQuery`를 소비하지 않음** (`useOutletContext` 미사용)
- 실시간 필터링 로직 없음
- 디바운스 처리 없음

## 수용 기준 (Acceptance Criteria)

- [ ] 검색바에 텍스트 입력 시 엔터 없이 실시간으로 필터링된다
- [ ] 공연명으로 검색이 가능하다
- [ ] 장소명으로 검색이 가능하다
- [ ] 대기 목록, 페스티벌, 콘서트 페이지에서 모두 동작한다
- [ ] 대시보드에서는 검색이 동작하지 않는다
- [ ] 검색 결과에서도 진행 예정/종료 분류가 유지된다
- [ ] 검색 결과 항목 클릭 시 정보 페이지로 이동한다
- [ ] 검색어 삭제 시 전체 목록으로 복원된다
- [ ] 검색 결과가 없으면 빈 상태 UI가 표시된다
