# 대기 목록 (Pending) — Requirements

> 크롤링/수기 등록된 공연 중 아직 게시되지 않은 공연 목록. 검토 후 정보를 수정하고 서비스에 게시할 수 있다.

**Figma**: [Admin 명세 — 대기목록](https://www.figma.com/design/LcVY1m4kQmMVDGaIPuiJJy/Admin-%EB%AA%85%EC%84%B8?node-id=28-748)

## User Flow

### A. 대기 목록 조회

1. `/pending` 접근 시 대기 중인 공연 리스트 표시
2. 각 항목에 공연 타입(F/C 아이콘), 제목, 날짜, 장소, "검토 필요" 뱃지 표시
3. 상단에 전체 대기 공연 수 합산 표시
4. 항목 클릭 → `/performance-editor/:id`로 이동 (정보 수정)
5. 더보기(`⋮`) 버튼 → 추가 액션 (삭제 등)
6. 빈 상태: "대기 중인 공연이 없습니다."

### B. 수기 등록

1. "수기 등록" 버튼 클릭 → `/performance-editor/new`로 이동
2. 상단 탭으로 페이지 전환 (기본 정보 / 상세 정보 / 라인업 / 타임테이블)

#### B-1. 기본 정보 탭

- **공연 분류**: 페스티벌 / 콘서트 선택
- **공연 기본 정보**: 공연명, 부제목 입력
- **공연 일정**: 캘린더로 시작일/종료일 선택
  - 종료일 < 시작일 불가 (시작일 = 종료일은 가능)
- **관람 제한**: 관람 등급, 러닝 타임 텍스트 입력
- **공연 예매 일정**: 텍스트 입력, 예매 일정 추가/삭제 가능 (2차, 3차 등)
- **예매처 선택**: 복수 선택 가능, 선택 시 URL 입력 가능

#### B-2. 상세 정보 & 미디어 탭

- **장소 정보**: 장소명, 주소 텍스트 입력
- **좌석 등급 및 가격**: 등급명, 가격 텍스트 입력
  - 가격 등급 추가/삭제 가능
- **메인 포스터 업로드**: JPG, PNG
- **로고 업로드**: 선택사항 (페스티벌은 필수), PNG
  - 타임테이블 랜딩 페이지 내에 노출됨

#### B-3. 라인업 탭 (페스티벌 전용)

- **스테이지 관리**: 스테이지명 텍스트 입력, 추가/삭제
  - 스테이지 삭제 시 해당 타임테이블 정보도 초기화됨
- **아티스트 검색**: 검색으로 아티스트 추가
  - `+` 버튼 또는 텍스트 필드 선택으로 추가 가능
- **타임테이블 편집 이동**: 버튼 클릭 시 타임테이블 편집 화면으로 이동

#### B-4. 타임테이블 탭 (페스티벌 전용)

- 공연 시작일/종료일 미설정 시 → 엠티 뷰 표시
- 설정된 경우 → 타임테이블 편집 가능

### C. 저장 및 게시

- 기본 정보 + 상세 정보 & 미디어 **모두** 입력해야 게시 가능
- 클릭 시 confeti 서비스에 공연 정보 업로드
- 이미 업로드된 상태에서 추가 정보 기입 후 업데이트 가능

### D. 삭제

- 클릭 시 해당 공연 정보 삭제
- 삭제된 공연 정보 별도 보관하지 않음

## 화면 구성

### 대기 목록 리스트

- **헤더**: "대기 중인 공연" 제목 + 설명 + "수기 등록" 버튼
- **리스트**: 각 항목에 타입 아이콘, 제목, 날짜•장소, "검토 필요" 뱃지, 더보기 버튼
- **빈 상태**: Clock 아이콘 + "대기 중인 공연이 없습니다."

### 수기 등록 / 정보 수정 (performance-editor)

- **탭 네비게이션**: 기본 정보 | 상세 정보 | 라인업 | 타임테이블
- **하단 액션**: 삭제 버튼 + 저장 및 게시 버튼

## API 연동

### 현재 (Mock)

```ts
import { PENDING_ITEMS } from '@shared/mocks';
```

### 목표 (Real API)

| Method | Endpoint                               | 설명                | END_POINT 상수                        |
| ------ | -------------------------------------- | ------------------- | ------------------------------------- |
| GET    | `/admin/performances/drafts`           | 대기 공연 목록 조회 | `END_POINT.GET_DRAFTS`                |
| POST   | `/admin/performances/drafts`           | 대기 공연 등록      | `END_POINT.POST_DRAFT`                |
| GET    | `/admin/performances/drafts/{draftId}` | 대기 공연 상세 조회 | `END_POINT.GET_DRAFT_DETAIL(draftId)` |
| PATCH  | `/admin/performances/drafts/{draftId}` | 대기 공연 수정      | `END_POINT.PATCH_DRAFT(draftId)`      |
| DELETE | `/admin/performances/drafts/{draftId}` | 대기 공연 삭제      | `END_POINT.DELETE_DRAFT(draftId)`     |

- Query keys: `DRAFT_QUERY_KEY.LIST()`, `DRAFT_QUERY_KEY.DETAIL(draftId)`

## 구현 노트

- 대기 목록 페이지: `pages/pending/pending-page.tsx`
- 수기 등록/수정은 `pages/performance-editor/` 페이지에서 처리
- "검토 필요" 뱃지: 크롤링된 공연 정보 모두에게 표시
- 공연 타입 아이콘: Festival → 보라색 'F', Concert → 초록색 'C'
- 검색: Layout에서 `searchQuery`를 Outlet context로 전달 (현재 대기 목록에서 필터링 미구현)
- 네비게이션: `PATH.PERFORMANCE_EDITOR.replace(':id', id)` 패턴 사용

## 수용 기준 (Acceptance Criteria)

### 리스트

- [ ] `/pending` 접근 시 대기 중인 공연 리스트가 표시된다
- [ ] 각 항목에 타입 아이콘, 제목, 날짜, 장소가 표시된다
- [ ] 크롤링된 공연에는 "검토 필요" 뱃지가 표시된다
- [ ] 항목 클릭 시 정보 수정 페이지로 이동한다
- [ ] 대기 항목이 없으면 빈 상태 UI가 표시된다
- [ ] "수기 등록" 버튼 클릭 시 신규 등록 페이지로 이동한다

### 수기 등록 / 수정

- [ ] 페스티벌/콘서트 타입 선택이 가능하다
- [ ] 공연 일정의 종료일이 시작일보다 빠를 수 없다
- [ ] 예매 일정을 추가/삭제할 수 있다
- [ ] 예매처를 복수 선택하고 URL을 입력할 수 있다
- [ ] 장소 정보, 좌석 등급/가격을 입력할 수 있다
- [ ] 포스터 이미지를 JPG/PNG로 업로드할 수 있다
- [ ] 페스티벌의 경우 로고 업로드가 필수이다
- [ ] 스테이지 추가/삭제가 가능하다 (삭제 시 타임테이블 초기화)
- [ ] 아티스트를 검색하여 추가할 수 있다

### 저장 및 게시

- [ ] 기본 정보 + 상세 정보 모두 입력해야 게시 가능하다
- [ ] 게시 후 추가 정보 업데이트가 가능하다
- [ ] 삭제 시 복구 불가능하다
